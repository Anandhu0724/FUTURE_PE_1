## 🧠 Intern Core Concept: Structured Context Injection

When building client interfaces, **never** ask an LLM to blindly *"write a website text."* You must train the AI using our three-layer context injection framework:

1. **The Core Context Anchor:** Feed the LLM an uncompromised identity profile (unique local differentiators, regional SEO targets, precise audience personas).
2. **Neuro-Linguistic Framing (NLF):** Force the AI to use behavioral psychology—shifting away from empty marketing clichés toward **Functional Restorations** (e.g., using our *"Stolen Capability Hook"*).
3. **Structured Design Constraints:** Direct the output formatting (UI labels, accordion data arrays, micro-copy) so it can be pasted directly into your visual component tree without rewriting.

## 🛠️ Fault-Tolerant Client-Side Architecture (Deployment SOP)
A common mistake interns make when deploying prototypes to static hosts like Netlify is hardcoding backend routes or server keys (`VITE_API_URL`), resulting in broken `404 Bad Gateway` errors during evaluation.

* **Our Framework Standard:** This codebase bypasses rigid server infrastructure dependencies by executing a **1-second artificial asynchronous network delay loop** directly on the client side. 
* **The Rule:** This allows the app to stay 100% stable, free to host, secure from API key leaks on public GitHub branches, and completely functional for client demos out-of-the-box.
